# Contributing to Winfr Pro

First off, thank you for considering contributing to **Winfr Pro**! It's people like you that make open-source such a great community.

### Reporting Bugs
- Use the **Bug Report** template on GitHub Issues.
- Provide a clear summary and steps to reproduce the issue.

### Suggesting Enhancements
- Check the [Roadmap](README.md#roadmap--future-plans) to see if it's already planned.
- Use the **Feature Request** template to explain your idea.

### Pull Requests
1. **Fork** the repository and create your branch from `main`.
2. Ensure the **CI** workflow passes (lint and build checks).
3. Follow the existing coding style (Standard React/TS and Rust best practices).
4. Update the documentation if your change adds new functionality.
